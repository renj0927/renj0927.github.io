一：是什么

  • 是一个声明式，用独立简单的代码，组合成复杂的UI界面。最重要的就是组件

    二：组件

    React.Component的子类
  • 所有的组件都相当于是React.Component的子类。每一个自定义组件前都要添加这句话。
  • 在react中，组件的本质就是一个函数。在return中添加想要返回的结构。
  • 大的应用中就可以将组件拆分成多个小的组件，嵌套在一起。
  • react组件都必须像纯函数一样保护他们的props不被修改。
1：创建组件的步骤:
  • 使用import语句引入react和RN相应的组件（RN自带的那种）
  • 函数===组件
  • 返回值就会渲染成一个react元素。
  • 如果想要在别的地方使用组件，需要将它暴露出去。
    2：一个组件建立到更新发生了什么
  • （ReactDOM.render()）调用ReactDOM.render()函数，并将传入作为参数。
  • （调组件，传参）react调用welcome组件，并将name作为props传入。
  • （元素作为返回值）welcome组件将 元素作为返回值
  • react dom 将dom节点高效的更新为 。

    二：JSX

  • JSX==JS+XML。 就是在组件中可以插入html的类似语法。
  • ? 只能有一个text标签吗
  • 可以像写js一样写一个框架，还可以设定子元素，然后包含很多的标签。这种写法就有点像js中的对象
  • 可以在JSX中插入用户输入的内容，可以防止注入攻击
    • 原理：react DOM 在渲染之前会进行转义，确保注入的都是自己明确编写的内容。 所有的内容都被转换成了字符串，这样就可以有效的防止XSS攻击。

三：state状态 & 生命周期

state
  • state就是一个状态。相当于vue中的Data，只要更新组件的state，就会根据新的state重新渲染。但是，这个state只是组件内部能够使用，如果想要在别的页面使用，就要进行传值。需要注意三点：
    • 1：state初始化—在组件的最初要使用一个constructor 来定义这个组件自己有的state。
    • 2： 更新state—使用setState来更新。(setState()是异步的，所以写的时候必须有回调函数。所以在准备数据再到更新的时候，会有一定的延时。调用这个就会调用render（）所以就会更新页面，但是延迟可能是个问题)
    • 3：调用different算法，判定是否要更新，这是他自有的判断。
  • 使用的场景：一般有用户输入，请求服务器，操作数据库等需要和后端数据打交道的地方，就需要state。一般就是构造几个无状态的组件。然后专门构建一个有状态的组件，用来进行传值。（就是别的组件不写state，专门有一个组件来取值和传值。就相当于是vue中的store，只不过这里是给他简化了写法。）
生命周期
  • 只有三个阶段，挂载，渲染，卸载。但是一般卸载不用，所以，只要关注挂载过程和更新过程就行。（其实就是react自己的api，直接调用就可以了，他就会在渲染的不同阶段调用。）
  • 挂载卸载过程：挂载就是组件状态的初始化，组件将相应的dom元素生成好后挂载到dom树的过程。
    • 1：componentWillMount()— 在render方法之前执行。可以执行setState，但是最终的渲染只有一次（？？？）（渲染前）
    • 2：componentDidMount()— 在render方法之后执行。渲染之后再取值，这样会不会就会造成一种卡顿，但是是不是可以将那种后面才跳出的页面写在这里。（渲染后）
    • 3：componentWillUnmount()— 卸载只有一个方法。一般会执行一些清理方法。（？？？ 时间回收、清除定时器）
  • 更新过程：父组件向下传递props或者组件自身执行setState时发生的一系列更新动作。
    • 1：shouldComponentUpdate(nextProps,nextState)— 接受的参数是更新的props和state。让开发可以增加必要的判断条件，需要时更新，不需要时不更新。返回值是一个boolean。
    • 2： componentWillUpdate（）和componentDidUpdate（）— 同渲染的过程。will要提供更新的props和state。而did要提供更新前的值。
    • 3：componentWillReceiveProps(nextProps)— 不行了！！！！

    七：循环渲染

    1： map和foreach
  • 相同点：
    • 都可以循环数组的每一项
    • 能循环获得的参数是item，index，arr。
    • 都可以在里面嵌套一个函数，来使用循环后的值。
  • 不同点：
    • map会循环遍历数组，并做一个返回。但foreach没有返回值。
    • foreach改变原始数组。map返回新数组。（所以在循环渲染时使用的是map，因为map有返回值。）

2:一定要注意的点

  • 一定要有key值！！！！！！！ key值最好是列表中的独一无二的字符串。（通常用数据的id或者索引值来做key。） key为什么是必须的 key值要放在循环中。

十：小杂项

1：ref

  • 是用来操纵react组件或DOM元素的接口。（就像是组件或元素的名字）
  • 使用方式：
    • string绑定：相当于用ref给一个dom节点起名字，之后就可以用this.ref.name来控制节点。但是最新版react已经不推荐使用了。可能会被遗弃。
    • react.CreateRef()：使用它来创建一个变量，然后可以直接使用ref来调用这个变量。
    • 函数形式：将子组件暴露给父组件，使得父组件能够调用子组件的方法。
  • 在react中，能使用state解决的事就不要使用ref。
